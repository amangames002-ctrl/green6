未來部署方向

你問到靜態還是動態，我的結論是：先靜態，後面視資料來源與更新頻率再升級。

Phase 1 · 純靜態頁

最適合現在。回測腳本在本機跑完後輸出 `json/html`，再直接部署到 `GitHub Pages / Cloudflare Pages / Vercel` 都行，成本最低、最穩。

Phase 2 · 靜態 + 自動更新

用 GitHub Actions 或 Cloudflare Workers Cron 每天收盤後跑一次回測，重新生成靜態頁，還是維持免後端。

Phase 3 · Hybrid 動態

只有在你真的要做篩選器、即時查詢、多標的互動比較時，才需要前端 + API。免費平台可再看 `CF / Vercel / Firebase`。

目前建議

這一版就是我對你現在需求的產品化理解。

先做靜態比較合理

  • 回測本身不是即時互動需求
  • 你每天收盤後更新一次就夠
  • 免費部署空間都能輕鬆支援

之後再升級成動態

  • 當你需要多標的、多策略切換
  • 需要篩選條件與日期範圍互動
  • 需要即時查詢或多人共用

這個 v2 跟前一版的差別

不是單純把內容塞更多，而是把它更像一個你真的會每天打開看的回測儀表板。
