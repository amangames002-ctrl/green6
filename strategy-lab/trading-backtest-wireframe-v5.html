訊號窗口

這一頁未來直接接 `check_entry()` 與即時 context。不是收盤報告，而是每天固定的兩個 BTC 訊號窗口。

Signal Windows
Primary Runtime Output

目前建議：等待窗口

正式上線後，這裡只會輸出三種狀態： `LONG`、`SHORT`、`NO TRADE`。 依據不是主觀判斷，而是當前小時是否剛好命中策略時段，以及 `trend_12h_pct` 是否符合條件。

Runtime Status IDLE

等待 `22:00` 或 `00:00` 台北時間的策略窗口。

今晚多單窗口 22:00 台北

對應 `14:00 UTC`，若 `trend_12h_pct > 0`，觸發 Strategy E 做多。

今晚空單窗口 00:00 台北

對應 `16:00 UTC`，若 `trend_12h_pct < 0`，觸發 Strategy C 做空。

觀望條件 No Trade

未到策略時段，或 `trend_12h_pct` 不符合該策略方向時，直接觀望。

入場價格 00.00

依 repo backtester / live executor，預設以訊號 bar 的 `close` 作為進場價。

風控規則 SL 1 ATR / TP 2 ATR

repo 目前採 `1.0 ATR` 停損、`2.0 ATR` 止盈，沒有額外技術指標濾網。

時間出場 6 bars / 6h

未達止盈止損時，持倉滿 `6` 根 `1H` bar 強制離場。

執行流程

這裡對應未來的實際排程，不靠臨場判斷。

  1. 1
    抓最新 1H bars
    從 Binance 取回 `BTCUSDT` 近期 `1H` OHLCV，建立 context。
  2. 2
    計算 context
    至少要有 `hour_utc`、`trend_12h_pct`、`ATR(14)`，這些都已在 repo 裡有對應實作。
  3. 3
    檢查策略窗口
    `22:00` 檢查 Strategy E，`00:00` 檢查 Strategy C；只有命中條件才發提醒。
  4. 4
    產生網站與通知
    把當前決策、SL / TP / 6h exit、push 狀態輸出到靜態頁與 Discord。
