Final Tab

核心原理

  • 不使用傳統技術指標，例如 RSI、MACD、布林帶與均線。
  • 以美股風險偏好、時段、盤中狀態作為主要交易依據。
  • 止盈與止損邏輯沿用文章策略，不在這個框架階段改動。
  • 即使未達止盈止損，只要滿 6 小時就強制離場。

風險管理

  • 不能只看勝率，還要同時看最大回撤與強制離場比例。
  • 若 Time Exit 比例過高，通常代表進場邏輯不夠集中。
  • 之後正式回測時必須納入手續費與滑價。
  • 若日後加入多標的，也要拆開看不同市場的 regime 表現。
靜態部署建議

現階段最合理：本機回測完成後輸出 HTML/JSON，再部署到 GitHub Pages、Cloudflare Pages 或 Vercel。這樣維護成本最低，也最不受技術棧限制。

何時升級成動態

只有當你真的需要前端篩選器、多人共用、即時查詢、多策略切換，才值得加 API 或資料庫。現在先用靜態版是對的。

這個 `v3` 是靜態展示版。互動只有前端 tab 切換、月份切換與每週逐次載入，占位資料全部為示意，目的是把未來產品的閱讀流程先定下來。
